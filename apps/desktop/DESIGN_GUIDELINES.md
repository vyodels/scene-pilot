# Desktop UI Design Guidelines

> 本文件是 **设计风格模板**，不是内容模板。  
> Codex / Claude 在生成任何 `apps/desktop` 的 UI 代码时都必须遵守本规范。  
> 下文出现的"候选人 / 投递记录 / 职位 / 沟通"等字段只是 **风格填充示意**，真实业务字段以当前页面需求为准；**禁止把示例里的字段名、文案、数量原样照抄到其他页面**。

---

## 0. 使用须知（给 AI）

1. 本文件只约束**视觉语言、布局骨架、组件形态、交互节奏**。
2. 业务字段、业务文案、表格列、表单项必须来自当前 Task 的需求或上游类型（`packages/shared`），**不要**从本文件抄示例字段。
3. 前端不得硬编码或 mock 业务字段值；候选人、JD、阶段、状态、数量、评分、渠道、时间线、审批等数据必须来自后端 API、共享契约或明确的用户输入。
4. 后端未提供的数据必须渲染为空、禁用或“待获取”，不得用示例值、合成值、随机值或兼容兜底假装已有数据；缺失字段应先记录并确认语义、来源和兼容策略，再新增后端字段或映射。
5. 每次生成新页面前，先在回答中用一句话声明："本页套用 DESIGN_GUIDELINES 的 `<骨架名>` + `<组件集合>`"，再开始写代码。
6. 如果业务需求与本规范冲突，先在 PR/回复中指出冲突点并建议更新本文件，**不要**私自偏离。
7. 冲突优先级：`业务正确性 > 本规范 > 个人审美`。

## 1. 风格总纲

- **定位**：中文 SaaS 工作台风格 —— 干净、信息密度偏高、亲和、轻微科技感。参考国内主流招聘 / CRM / 工单类产品的排版节奏。
- **关键词**：白底为主 · 青蓝主色点缀 · 扁平无重阴影 · 紧凑圆角卡片 · 中文优先排版 · 克制动效。
- **明确禁止**：暗黑主题、毛玻璃、霓虹、紫色渐变 Hero、Neo-brutalism、视差滚动、粒子/光晕背景、emoji 装饰、AI 味渐变按钮。

## 2. 设计 Token（唯一事实来源）

所有颜色/间距/圆角必须走 CSS 变量，**严禁在组件里硬编码 hex 或 px 数值**（除非该值来自下列 token）。建议落地到 `src/styles/tokens.css` 并在根样式引入。

```css
:root {
  /* —— 品牌主色 —— */
  --brand-primary:         #00BEBD;
  --brand-primary-hover:   #13D0CF;
  --brand-primary-pressed: #009E9D;
  --brand-primary-soft:    #E6F9F9;

  /* —— 中性文字 —— */
  --text-primary:     #222933;
  --text-regular:     #495266;
  --text-secondary:   #7A8499;
  --text-placeholder: #AAB2BF;
  --text-inverse:     #FFFFFF;

  /* —— 背景 —— */
  --bg-page:         #F5F6F8;
  --bg-card:         #FFFFFF;
  --bg-hover:        #F2F7F8;
  --bg-sider:        #2B2F3A;
  --bg-sider-hover:  rgba(255,255,255,0.06);
  --bg-sider-active: #00BEBD;

  /* —— 边与线 —— */
  --border-line:  #EBEDF0;
  --border-input: #D9DCE1;
  --border-focus: #00BEBD;

  /* —— 语义色 —— */
  --success: #00B86B;
  --warning: #FFA940;
  --danger:  #FF4D4F;
  --info:    #4B8BF5;

  /* —— 间距（4 的倍数）—— */
  --space-1: 4px;  --space-2: 8px;  --space-3: 12px;
  --space-4: 16px; --space-5: 20px; --space-6: 24px;
  --space-8: 32px; --space-10: 40px;

  /* —— 圆角 —— */
  --radius-xs: 4px;
  --radius-sm: 6px;
  --radius-md: 8px;
  --radius-lg: 10px;
  --radius-full: 9999px;

  /* —— 阴影（极少使用）—— */
  --shadow-pop: 0 4px 16px rgba(31,35,41,0.08);

  /* —— 字体 —— */
  --font-sans: "PingFang SC","Microsoft YaHei","HarmonyOS Sans SC",
               "Source Han Sans CN", system-ui, -apple-system,
               "Helvetica Neue", Arial, sans-serif;
  --font-mono: "SF Mono","JetBrains Mono", Menlo, Consolas, monospace;
}
```

**用色比例**：页面面积约 **80% 白/浅灰 + 15% 中性文字 + 5% 主色**。主色只服务"行动召唤"与"当前选中"，不做大面积铺底。

## 3. 字体与排版

- 字体栈统一使用 `var(--font-sans)`。**禁止**引入 Inter / Roboto / Arial 作为主字体。
- 字号阶梯（px / 字重）：
  - 页面主标题 `18 / 600`
  - 区块标题 `16 / 600`
  - 正文默认 `14 / 400`
  - 次要说明 `13 / 400`
  - 辅助/时间戳 `12 / 400`
- 行高：正文 `1.6`，标题 `1.4`。
- 字重只用 `400 / 500 / 600`，**不用 700+**。
- 强调只用加粗，不用换色。
- 中英文之间交由浏览器自然排版处理，**不手动加空格**。

---

## 4. 间距 / 圆角 / 阴影 / 分割线

- 间距一律 4 的倍数，通过 `--space-*` 使用。卡片内边距默认 `var(--space-4)`，卡片之间 `var(--space-3)`。
- 圆角只使用 `--radius-xs/sm/md/lg` 四档；头像用 `--radius-full`。禁止出现 `16px+` 大圆角造成"玩具感"。
- 阴影几乎不用。**只有浮层**（下拉、气泡、Toast）才允许 `--shadow-pop`。卡片一律用 `1px solid var(--border-line)` 表达层级。
- 分割线一律 `1px solid var(--border-line)`，不用虚线、不用双线。

## 5. 全局骨架（所有业务页面通用）

```
┌──────────────────────────────────────────────┐
│  TopNav  高 48px / 白底 / 下 1px 边            │
├──────┬───────────────────────────────────────┤
│      │                                       │
│ Side │          Page Content                 │
│ Nav  │   bg: var(--bg-page)                  │
│ 72px │   padding: var(--space-5)             │
│ 深色 │                                       │
└──────┴───────────────────────────────────────┘
```

### 5.1 SideNav（左，`72px`，`--bg-sider`）

- 纵向图标 + 2 字中文标签；项高 `64px`；图标 `24px` 描边；不折叠。
- 选中态：整块 `--bg-sider-active`，图标与文字 `--text-inverse`。
- hover：`--bg-sider-hover`。
- 底部固定：Logo + 角色/环境徽章（小字浅灰）。
- **字段示意（非强制）**：首页 / 工作台 / 搜索 / 沟通 / 资料库 / 我的 —— 真实菜单以业务为准。

### 5.2 TopNav（顶，`48px`，白底）

- 左：一级 Tab，选中项 **底部 2px 主色下划线**，无底色填充。
- 中：可选全局搜索框（仅业务需要时出现）。
- 右：通知 / 消息 / 头像 / 下拉，图标统一描边 `20px`。

### 5.3 Page Content 常见骨架（任选其一，不是强制清单）

| 骨架名 | 适用场景 | 结构要点 |
|---|---|---|
| `ListFilterPage` | 带筛选的列表/搜索 | 顶部多行 chip 过滤器 → 结果工具栏 → 卡片列表 |
| `TablePage` | 后台管理/表格 | 顶部操作条 → 表格 → 底部分页 |
| `DetailSplitPage` | 主从/会话 | 左 `280px` 列表 + 中弹性主区 + 右 `320px` 详情 |
| `FormPage` | 设置/向导 | 左二级菜单 + 右侧表单卡片 |
| `DashboardPage` | 数据概览 | KPI 网格 + 待办列表，禁止彩色渐变卡 |

**挑选骨架的原则**：先判断页面信息角色（浏览？操作？沟通？配置？概览？），再套骨架；不要为了统一把所有页面都塞进同一个骨架。

### 5.4 Compact Chain Filter（紧凑链式筛选，投递记录漏斗 / 投递记录跟进基线）

当页面需要同时表达“筛选上下文 + 漏斗阶段/状态链”时，统一采用紧凑链式风格，不再为每个页面单独发明一套顶部条样式。

- 顶部筛选条使用一行 `kanban-filter-row`，只放当前页面真正生效的筛选器。
- 阶段/状态节点必须是**单行** `名称-数量`，禁止拆成上下两行。
- 字号统一 `11px–12px`，参考页面右上角 `后端已连接` 这一档，不允许出现大数字单独放大。
- 节点使用浅底色 + 1px 细边框 + 胶囊圆角，视觉密度高，允许换行，但要遵守“当前行排满再换下一行”。
- 节点之间必须有明确箭头 `→`，不是靠空白暗示顺序。
- 选中态必须是**整块节点**高亮：浅主色底 + 完整边框；不能只保留底部一条色线。
- 失败分支、旁路节点要挂靠在主节点旁或下方，不能散落成一组无关系标签。
- 页面如果同时存在“聚合筛选项”和“阶段链条”，两者的字号、圆角、边框、浅底色强度必须保持同一套视觉层级。
- 新页面需要复用这套模式时，优先沿用现有投递记录漏斗 / 投递记录跟进的 `kanban-filter-row`、`funnel-stage-bar`、`status-chain` 结构，不新造并行风格。

## 6. 基础组件规范

### 6.1 Button

| 类型 | 背景 | 文字 | 边框 | 用途 |
|---|---|---|---|---|
| Primary | `--brand-primary` | `--text-inverse` | 无 | 核心行动，每视图 ≤ 1–2 个 |
| Default | `--bg-card` | `--text-primary` | `--border-input` | 次级操作 |
| Text | 透明 | `--brand-primary` | 无 | 行内/链接型 |
| Danger | `--bg-card` | `--danger` | `--danger` | 破坏性 |

- 高度：`大 36 / 默认 32 / 小 28`，水平 padding `var(--space-4)`。
- 圆角 `--radius-sm`，**不用胶囊**。
- hover：Primary → `--brand-primary-hover`；Default → 边框与文字换 `--brand-primary`。
- 禁用：底 `#F2F3F5`，字 `#BBBFC4`，无阴影。

### 6.2 Input / Form

- 高 `32px`，圆角 `--radius-sm`，边 `--border-input`。
- 聚焦：边 `--border-focus` + `0 0 0 3px rgba(0,190,189,.15)`。
- Label 置上方，14px 常规字重。
- 错误：边 `--danger` + 下方 `12px` 红字提示。
- 表单项纵向间距 `var(--space-4)`。

### 6.3 Card

- 白底 + `1px` 细边 + `--radius-md`，**不加阴影**。
- 头部：左 `var(--space-4)` 标题，右操作区，与内容之间 `1px` 分隔线。

### 6.4 Tag

- 高 `22px`，圆角 `--radius-xs`，字号 `12px`，水平 padding `var(--space-2)`。
- 变体：
  - **中性**：底 `#F2F3F5`，字 `--text-regular`
  - **主色**：底 `--brand-primary-soft`，字 `--brand-primary`
  - **语义**：底为对应语义色 10% 透明，字为语义色
- 多个 Tag 间距 `var(--space-1)~var(--space-2)`。
- Tag **只承载属性、不承载操作**（别把 Tag 当按钮用）。

### 6.5 Avatar

- 尺寸 `32 / 40 / 48 / 64`，圆形。
- 可选右下 `10px` 在线状态点。
- 加载失败：首字 + 稳定中性底色（同一用户颜色固定）。

### 6.6 Table

- 表头 `#FAFBFC`；行高 `48px`；行 hover `--bg-hover`。
- 边框只有外框 + 行间 `1px` 横线，**不画竖线**。
- 操作列使用 Text 按钮，按钮之间用 `|` 分隔。
- 空态用居中插画（描边单色）+ 说明 + Primary 按钮。

### 6.7 Modal / Drawer / Popover

- Modal 默认宽度 `480 / 640 / 960` 三档，圆角 `--radius-md`，头/体/尾三段式。
- Drawer 默认宽度 `400 / 560`，右侧进入。
- Popover 用 `--shadow-pop`，箭头不强制。

### 6.8 图标

- 描边风格（Tabler / Lucide 同类），线宽约 `1.75`。
- 默认 `20px`，工具栏 `18px`，侧栏 `24px`。
- 颜色默认 `--text-regular`，选中/hover `--brand-primary`。**禁止多色、渐变、3D 图标**。

## 7. 复杂场景骨架：双列 / 三列工作台

> 这是本站出现频率最高的"沟通 / 会话 / 主从详情"类页面的视觉骨架。下面字段纯粹是风格示意，**真实字段以业务模型为准**。

```
┌──────── Left 列表 (~280px) ────────┬──────── Main 主区 (flex) ────────┬──── Right 详情 (~320px) ────┐
│ 搜索框                              │ 顶部：主体标题 + 操作按钮组         │ 头像大卡                      │
│ Tab（全部/未读/…）                  │ 次行：关键信息条（以 | 分隔字段）  │ 分区：基础信息                │
│ Item × N：                          │                                    │ 分区：关键属性 Tag 群         │
│   头像 / 主标题 / 时间 / 预览 / 红点 │ 内容区：时间线 or 消息流 or 卡片组 │ 分区：历史/时间线             │
│                                     │                                    │ 分区：AI 亮点（浅色主色卡）   │
│                                     │ 底部：输入/操作区                  │ 底部：主操作按钮组            │
└─────────────────────────────────────┴────────────────────────────────────┴──────────────────────────────┘
```

- 左列 Item 高 `72px`；选中态整行 `--brand-primary-soft` + 左 `3px` 主色竖条；hover `--bg-hover`。
- 主区若为消息/时间线：
  - 日期/系统提示居中灰胶囊。
  - 左右对照气泡：对方白底 `1px` 边，自身主色底白字，圆角 `--radius-lg`，最大宽约主区 `60%`，相邻同源气泡间距 `var(--space-1)`，异源 `var(--space-4)`。
- 底部输入/操作区至少 `160px` 高：
  - 顶部一行小号工具条（图标 18 + 文字 12 灰）；具体条目由业务决定。
  - 中部 Textarea 无边框，placeholder 给出快捷键提示。
  - 右下 Primary 发送；聚焦时整个输入区顶边变 `--border-focus`。
- 右列详情使用"无框分区 + 区块标题 + 细分割线"组织；避免嵌套卡中卡。

> 再次提醒：上面的"Tab 名、字段名、分区名"都是 **占位示意**，不要照抄到其它业务页面。

## 8. 动效

- 克制。标准过渡 `150ms ease-out`，弹层 `200ms ease-out`。
- 消息/列表新增项：`opacity 0→1 + translateY(4px→0)`，`160ms`。
- 路由切换：仅内容淡入 `120ms`，无转场。
- **禁止**：bounce、弹簧、长路径、光晕、视差、鼠标跟随。

---

## 9. 可访问性与交互默认

- 焦点可见：`outline: 2px solid var(--border-focus); outline-offset: 2px;`。
- 正文对背景对比度 ≥ 4.5:1。
- 键盘：`Tab` 可循环、`Esc` 关弹层、`Enter` 提交表单/发送、`⌘/Ctrl+K` 打开全局搜索（若页面支持）。
- Toast：顶部居中，白底 + 左侧语义色图标，持续 `2s`，不阻塞操作。
- Loading：顶部 `2px` 主色进度条；局部用骨架屏（`#F2F3F5 → #FAFBFC` 线性渐变）。
- 空态：居中描边插画 + 14 灰说明 + 一个 Primary 按钮。

## 10. 技术落地约定（`apps/desktop`）

- 颜色/间距/圆角/字体必须通过 `tokens.css` 暴露的 CSS 变量引用，**PR 中出现新的 hex 值将被拒绝**；如需新增，先改本文件与 token 文件。
- 可复用原子组件放 `src/components/ui/`；业务组合放 `src/features/<domain>/components/`；页面容器统一用 `<AppLayout>`（SideNav + TopNav）包裹。
- 命名：组件 `PascalCase`，hooks `useXxx`，样式文件与组件同名。
- 任何新页面的第一次提交都应：
  1. 指明使用的骨架（第 5.3 节）；
  2. 列出复用的原子组件；
  3. 附一张截图或文字线框（PR 描述中）。
- 若 UI 涉及共享数据结构，类型放 `packages/shared`。

## 11. AI 自检清单（生成前必过）

每次产出 UI 代码前，在内部过一遍以下 13 项，任一不满足就重做：

1. 颜色是否全部走 CSS 变量？有没有硬编码 hex？
2. 主色使用是否克制（全页 ≤ 约 5% 面积）？
3. 字体是否为 PingFang SC 系？是否避开了 Inter / Roboto / Arial？
4. 字号/字重是否只使用第 3 节列出的阶梯？
5. 间距是否全部是 4 的倍数并走 `--space-*`？
6. 圆角是否只落在 `--radius-xs/sm/md/lg/full`？
7. 是否完全没有阴影（仅浮层除外）、没有渐变、没有毛玻璃？
8. 是否套用了第 5 节的全局骨架（SideNav + TopNav + ContentArea）？
9. 是否为该页面挑选了合适的 5.3 骨架，而不是生搬三列布局？
10. Primary 按钮数量是否 ≤ 1–2 个？
11. 业务字段/文案是否来自真实需求，而非照抄本文件示例？
12. 业务字段值是否全部来自后端 API / 共享契约 / 用户输入？缺失值是否为空、禁用或“待获取”，而不是 mock / 合成 / 兜底假数据？
13. 动效是否只使用第 8 节允许的几种？

---

## 12. 更新规则

- 本文件是 **设计事实来源**。任何对 token、骨架、组件形态的改动都必须先改它，再改代码。
- 修改本文件的 PR 必须在描述里写清楚"改了什么 / 为什么 / 影响哪些页面"。
- 旧页面可以渐进迁移；**新页面不允许绕过本规范**。
